# 소셜 카지노 앱 기획서 (v0.1)

> 작성일: 2026-06-26
> 한 줄 요약: 실제 환전이 없는 **가상 화폐 기반 소셜 카지노**. 첫 게임은 **바카라**, **모바일(iOS/Android) + 웹** 동시 지원.

---

## 1. 제품 개요

| 항목 | 내용 |
|---|---|
| 제품 성격 | 소셜 카지노 (가상 화폐, 실제 환전·출금 없음) |
| 핵심 게임 | 바카라 (1차 출시), 이후 슬롯/블랙잭 확장 |
| 플랫폼 | 모바일 앱 + 웹 (공용 백엔드) |
| 수익 모델 | 가상 코인 인앱결제(IAP), 리워드 광고, 시즌패스 |
| 타깃 | 카지노 게임을 부담 없이 즐기려는 캐주얼 유저 (실제 도박 X) |

### 왜 소셜 카지노인가 (포지셔닝)
- **법적 안전성**: 코인↔현금 환전이 없으면 한국 사행성 규제(게임산업법) 부담이 크게 줄어듦.
- 단, **출시 전 법률 검토 필수** — "환전 없음"을 코드·약관·운영 모두에서 일관되게 보장해야 함. (예: 코인을 외부에서 사고팔 수 있게 두면 안 됨)

---

## 2. 핵심 원칙 (기술/기획 공통)

1. **게임 결과는 100% 서버에서 결정한다.** 클라이언트는 표시/연출만. (조작·치팅 방지의 핵심)
2. **코인은 환전 불가.** 구매·획득만 가능, 현금화·유저 간 송금 불가.
3. **공정성(RNG)**: 서버 RNG는 검증 가능하게 설계 (provably fair 또는 감사 로그).
4. **클라이언트는 얇게, 서버는 단일 진실 공급원(Single Source of Truth).**

## 3. 시스템 아키텍처

```
┌─────────────┐   ┌─────────────┐
│ 모바일 앱    │   │  웹 앱      │   ← 클라이언트 (연출/UX만)
│ (Flutter)   │   │ (Next.js)   │
└──────┬──────┘   └──────┬──────┘
       │   REST + WebSocket   │
       └──────────┬───────────┘
            ┌──────▼──────┐
            │  API 게이트웨이 │
            └──────┬──────┘
        ┌──────────┼──────────────┐
   ┌────▼────┐ ┌───▼────┐  ┌──────▼──────┐
   │ 게임엔진 │ │ 지갑/   │  │ 유저/인증    │
   │(바카라) │ │ 코인서비스│  │ 소셜        │
   └────┬────┘ └───┬────┘  └──────┬──────┘
        └──────────┼──────────────┘
              ┌─────▼─────┐
              │  DB / 캐시  │  (PostgreSQL + Redis)
              └───────────┘
```

### 추천 기술 스택
- **클라이언트(모바일)**: Flutter (iOS/Android 단일 코드, 카드/칩 애니메이션에 강함)
- **클라이언트(웹)**: Next.js (React) — Flutter Web 대신 분리 추천 (SEO/로딩/유지보수 유리)
- **백엔드**: Node.js(NestJS) 또는 Go — 실시간 게임엔 WebSocket 필수
- **실시간**: WebSocket (베팅/결과/관전 동기화)
- **DB**: PostgreSQL(거래·유저), Redis(세션·게임상태·랭킹)
- **결제**: Apple/Google IAP + 웹은 PG사 연동

> 모바일/웹 클라이언트를 나누되 **백엔드·게임룰은 단일 코드**로 공유하는 게 핵심. 룰이 두 군데로 갈라지면 버그·불공정 발생.

## 4. 바카라 게임 설계 (1차 게임)

### 게임 룰 (Punto Banco 표준)
- 플레이어(Player) / 뱅커(Banker) / 타이(Tie) 중 베팅
- 각 2장 분배 → 합의 끝자리가 9에 가까운 쪽 승
- 3번째 카드 룰(드로우 규칙) 표준 적용
- 배당: Player 1:1, Banker 1:0.95(5% 커미션), Tie 8:1

### 게임 흐름 (서버 권위)
```
1. 라운드 시작 → 베팅 타임 (예: 15초)
2. 클라이언트: 베팅 요청 전송 (Player/Banker/Tie + 금액)
3. 서버: 코인 차감 검증 → 베팅 확정
4. 베팅 마감 → 서버 RNG로 카드 결정 → 룰 적용 → 결과 산출
5. 서버: 정산(코인 지급) → 결과 브로드캐스트
6. 클라이언트: 카드 오픈 애니메이션 (이미 정해진 결과 연출)
7. 결과·기록(로드맵) 표시 → 다음 라운드
```

### 핵심 화면
- 베팅 테이블 (Player/Banker/Tie 영역, 칩 선택)
- 카드 분배 애니메이션
- **로드맵(점수표)**: 빅로드/비드빈 등 — 바카라 유저가 매우 중시하는 기능
- 베팅 히스토리 / 현재 잔액

### 모드
- **싱글(봇 딜러)**: 1차 출시 — 구현 단순, 네트워크 부담 적음
- **멀티 테이블(라이브)**: 2차 — 여러 유저 동시 베팅 + 관전 (WebSocket 동기화)

## 5. 코인 / 경제 시스템

| 요소 | 설명 |
|---|---|
| 골드(플레이 코인) | 베팅에 사용. IAP·광고·일일보상으로 획득 |
| 젬(프리미엄 화폐) | 골드 구매, 부스터, 한정 아이템용 (선택) |
| 일일 보너스 | 접속 시 무료 코인 (리텐션 핵심) |
| 파산 방지 | 잔액 0이 되면 무료 코인 지급 (이탈 방지) |
| 리워드 광고 | 광고 시청 시 코인 — 비결제 유저 수익화 |

> **경제 밸런싱**이 소셜 카지노 수익의 핵심: 무료 코인은 "계속 놀 수 있게", 결제는 "더 크게 베팅/과시"로 유도. 환전이 없으므로 코인은 순수 엔터테인먼트 재화.
