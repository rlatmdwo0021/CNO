# 화면 구성 & 멀티룸 기획 (v0.2)

> 결정: ① 방 = **고정 프리셋(한도별)** · ② 로그인 = **매 실행 시 ID/비밀번호 입력**(자동로그인 없음)

## 1. 화면 흐름
```
[로그인/회원가입]  →  [로비: 게임 선택]  →  [방 선택]  →  [게임 테이블]
  ID + 비밀번호         바카라(운영중)         한도별 방 목록    (현재 바카라 화면)
  매 실행 시 입력        슬롯·블랙잭(준비중)     인원/상태 표시     ← 뒤로 = 방 목록
                        잔액 표시
```
- 앱을 켜면 **항상 로그인 화면**. 토큰을 기기에 저장하지 않음.
- 로그인 성공 시 세션 토큰은 **메모리에만**(끊김 재연결용), 앱 종료 시 사라짐 → 다음 실행 땐 다시 로그인.

## 2. 로그인 (ID/비밀번호)
- **회원가입**: 아이디(고유) + 비밀번호 + 표시이름 → 계정 생성
- **로그인**: 아이디/비밀번호 검증 → 기존 잔액·정보 복원
- 비밀번호는 **scrypt 해시로만 저장**(Node 내장, 추가 의존성 0). 평문 저장·로깅 안 함.
- 계정 저장소 필드: `playerId, username(고유), passwordHash, name, createdAt`
- 세션: 로그인 성공 → 랜덤 토큰 발급(서버 메모리 세션맵 `token→playerId`). 소켓 인증은 이 토큰으로. 서버 재시작 시 세션 만료 → 앱은 로그인 화면으로 복귀.

## 3. 로비 (게임 선택)
- 게임 레지스트리 구조: `[{id:'baccarat', name:'바카라', status:'live'}, {id:'slots', status:'soon'}, ...]`
- 지금은 바카라만 `live`, 나머지는 "준비중"(비활성 카드).
- 상단에 **잔액** 표시(전역, 모든 방 공유).

## 4. 방 선택 (멀티룸)
바카라 고정 프리셋 방(한도별):

| 방 | 이름 | 최소 | 최대 |
|---|---|---|---|
| 🟢 | 초보 | 10 | 500 |
| 🔵 | 일반 | 100 | 5,000 |
| 🟣 | VIP | 1,000 | 50,000 |

- 방마다 **독립된 게임루프 + 슈 + 로드맵**. 베팅 타이밍은 동일(베팅 8s).
- 방 목록 항목: 이름 · 한도 · 현재 인원 · 현재 상태(베팅중/딜링중).
- **지갑은 전역 1개** → 어느 방에서 베팅하든 같은 잔액에서 차감.
- 빈 방은 루프를 돌리되(로드맵 연속성), 리소스가 문제되면 "첫 입장 시 시작 / 마지막 퇴장 시 일시정지"로 최적화 가능(후순위).

## 5. 백엔드 구조 변화
```
기존:  server ── (단일) GameLoop ── Table ── Round ── engine
                                      └ wallet(전역)
목표:  server ── RoomManager ── Room#1 ── GameLoop ── Table ── Round ── engine
                              ├ Room#2 ── GameLoop ── ...      └ roadmap(방별)
                              └ Room#3 ...
                  wallet(전역, 모든 방 공유)   auth(계정/세션)
```
- **RoomManager**: 방 여러 개 관리. 소켓을 특정 방에 구독시키고, 방 단위로 이벤트 브로드캐스트.
- 소켓은 한 번에 한 방에만 입장. 방 전환 = leave 후 join.

## 6. 프로토콜 변경 (요약)
**클라이언트 → 서버**
- `register {username, password, name?}`
- `login {username, password}`
- `listRooms {gameId}`
- `joinRoom {roomId}` / `leaveRoom`
- `bet {betType, amount}`  ← 현재 입장한 방에 적용

**서버 → 클라이언트**
- `session {playerId, name, token, balance, games[]}`  / `authError {message}`
- `rooms {gameId, rooms:[{id,name,minBet,maxBet,players,phase}]}`
- `roomJoined {roomId, limits, phase, endsAt?, roadmap}`
- `open / locked / settled / bet`  ← 전부 `roomId` 포함, 해당 방 구독자에게만
- `betAck`, `balance`(전역), `error`

## 7. Flutter 화면
- `AuthScreen` — 로그인/회원가입 토글 (username, password, name)
- `LobbyScreen` — 잔액 + 게임 그리드
- `RoomListScreen` — 바카라 방 목록(한도/인원/상태)
- `TableScreen(roomId)` — 현재 테이블 UI를 방 단위로, 뒤로가기로 방 목록 복귀
- 연결/메시지는 `GameService` 한 곳에서 처리하고 화면 상태를 노출.

## 8. 만드는 순서
1. **Phase 1 — 로그인 개편**: AccountStore/AuthService(id/pw, scrypt) + 프로토콜 + `AuthScreen`
2. **Phase 2 — 멀티룸 백엔드**: RoomManager + 방 단위 프로토콜 + 프리셋 방 3개
3. **Phase 3 — Flutter 내비게이션**: 로비 + 방목록 + 방별 테이블
4. **Phase 4 — 다듬기 + 재배포(Render) + APK 재빌드**

## 9. 영향/주의
- 프로토콜이 바뀌므로 **백엔드 재배포 + APK 재빌드** 필요(자동 배포라 git push면 됨).
- 기존 익명 토큰 계정은 폐기(프로토타입). 새 id/pw 체계로 전환.
- Render 무료 콜드스타트(첫 접속 30~50s)는 그대로. 끊기면 앱은 로그인 화면으로.
