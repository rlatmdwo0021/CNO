# 라이브 딜러 하이브리드 — 영상 제작 스펙

딜러는 **미리 제작한 영상 클립**(AI 생성 또는 실제 촬영)으로 보여줍니다.
**카드 면은 영상에 들어가지 않습니다** — 실제 카드(서버 RNG가 결정)는 앱이 영상 위에
얹어서, 딜러의 손동작에 맞춰 합성합니다. 이렇게 하면 하나의 영상으로 가능한 모든 패에
재사용하면서도 항상 실제 게임 결과와 일치합니다.

따라서 모든 클립은 **카드 자리가 비어 있는 상태**로 제작해야 합니다 (딜러는 *빈 카드* 혹은
*뒷면만 있는 카드*를 놓고/뒤집고, 카드 면은 앱이 그립니다).

## 카메라 & 프레이밍 (모든 클립에서 반드시 동일해야 함)
- **고정 카메라. 줌·팬·컷 없음.** 모든 클립이 매끄럽게 이어지도록 같은 렌즈, 같은 높이,
  같은 화이트밸런스 사용.
- 권장: 딜러의 어깨 너머 3/4 탑다운 시점. 테이블 펠트가 화면 하단 2/3을 채우고, 딜러의
  상체/손이 상단에 위치.
- 해상도 1080×1920 (세로, 폰 앱과 동일) 또는 1080p로 찍고 크롭.
- 30fps. 노출/화이트밸런스 고정.
- **카드 슬롯 기준점:** 6개 카드 위치에 작고 제거 가능한 펠트 마커(또는 테이프 십자)를
  표시 — 플레이어 1/2/3 (왼쪽), 뱅커 1/2/3 (오른쪽). 이 위치의 픽셀 좌표를 한 번 측정하고,
  후반 작업에서 마커를 지우거나 눈에 띄지 않게 유지.

## 동일한 딜러, 동일한 룩 (클립 간 일관성)
- 한 사람, 같은 의상, 같은 조명, 같은 테이블. (AI의 경우: 캐릭터 레퍼런스 이미지 + 고정
  시드를 잠그고 같은 레퍼런스에서 모든 클립 생성.)

## 클립 목록 (각각 별도 파일, 루프/원샷 표시대로)

| id | 길이 | 루프? | 내용 |
|----|------|-------|------|
| `idle` | 6–10초 | **루프** | 딜러가 앉아서 편안하게, 가끔 작은 움직임. 손은 가만히. 베팅 시간 동안 사용. 매끄럽게 루프되어야 함(시작 포즈 == 끝 포즈). |
| `place_bets` | 2–3초 | 원샷 | 딜러가 테이블 쪽으로 손짓하며 "Place your bets" 후 idle 포즈로 복귀. |
| `no_more_bets` | 1–2초 | 원샷 | "더 이상 베팅 없음" 손짓. 딜 시작 포즈 근처에서 끝남. |
| `deal_4` | 3–4초 | 원샷 | 카드 4장을 **뒷면으로** 순서대로 딜: 플레이어#1, 뱅커#1, 플레이어#2, 뱅커#2. 각 카드를 표시된 슬롯에 놓음. 카드마다 손이 프레임을 벗어났다가/중립으로 돌아옴. |
| `flip_p1`,`flip_b1`,`flip_p2`,`flip_b2` | 각 ~0.8초 | 원샷 | 해당 슬롯의 카드 한 장을 뒤집음(손이 빈 카드를 뒤집는 동작). 앱이 이 동작 중에 뒷면→앞면 크로스페이드. 타이밍이 고정이라면 `flip_all` 하나로 통합 가능. |
| `deal_p3` / `deal_b3` | 각 ~1.2초 | 원샷 | 플레이어/뱅커 3번 슬롯에 선택적 3번째 카드를 딜 + 뒤집기. |
| `win_player` / `win_banker` / `win_tie` | 2–3초 | 원샷 | 딜러가 이긴 쪽을 가리키며 작은 리액션. 선택. |

> 최소 구성: `idle`, `deal_4`, `flip_all` (또는 개별 flip 4개), `deal_p3`, `deal_b3`.
> 나머지는 디테일/연출용.

## 카드 오버레이 계약 (앱이 각 클립에서 필요로 하는 것)
카드가 등장하는 모든 클립마다 다음이 필요합니다:
1. 그 클립 프레임에서 각 카드 슬롯의 **픽셀 좌표 + 회전각 + 크기**.
2. 각 뒤집기의 "절반 지점" **타임스탬프**(클립 시작 기준 ms) — 카드 면이 교체되어야 하는
   순간. `deal_4`의 경우, 뒷면 카드가 각각 테이블에 놓이는 타임스탬프.

각 클립 옆에 작은 JSON으로 전달, 예시:
```json
{
  "clip": "flip_all",
  "durationMs": 3200,
  "slots": {
    "p1": { "x": 280, "y": 1180, "w": 150, "h": 210, "rotDeg": -4, "flipMidMs": 400 },
    "b1": { "x": 700, "y": 1180, "w": 150, "h": 210, "rotDeg":  3, "flipMidMs": 1200 }
  }
}
```
앱은 기존 2D 카드 에셋을 이 슬롯 위치에 배치하고(테이블 각도에 맞춰 약간의 원근 기울임),
`flipMidMs`에 뒷면→앞면을 교체합니다.

## AI 생성 프롬프트 (시작점 — 툴에 맞게 조정)
> "Fixed locked-off camera, portrait 9:16, over-the-shoulder 3/4 view of an elegant
> baccarat dealer in a black vest and white shirt at a green felt casino table, warm
> casino lighting, dark luxurious background. The dealer's hands deal and handle
> **plain blank playing cards with no printed faces** on the green felt. Consistent
> character, consistent outfit, consistent lighting. No text, no card faces, no
> numbers on the cards. Smooth natural hand motion. Static background, no camera
> movement."

이후 동작마다 프롬프트 하나씩(idle / 빈 카드 4장 딜 / 빈 카드 뒤집기 등)을 **같은 캐릭터
레퍼런스 + 같은 시드**로 생성.

(한글 참고 번역) 위 프롬프트의 요지: "고정된 카메라, 세로 9:16, 검은 조끼·흰 셔츠를 입은
우아한 바카라 딜러를 어깨 너머 3/4 시점으로. 초록 펠트 카지노 테이블, 따뜻한 카지노 조명,
어둡고 고급스러운 배경. 딜러의 손이 **면이 인쇄되지 않은 완전히 빈 카드**를 다룸. 캐릭터·의상·
조명 일관. 글자·카드 면·숫자 없음. 자연스러운 손동작. 배경 고정, 카메라 이동 없음."
> AI 툴 대부분이 영어 프롬프트에서 더 잘 동작하므로 영어 원문을 사용하고, 한글은 참고용입니다.

## 호스팅
- 처음엔 클립을 **앱에 번들로 포함**(짧고 잘 압축한 H.264/H.265 몇 개). 전체 용량이 커지면
  CDN/Render 정적 호스팅으로 옮겨 `video_player`의 네트워크 소스로 스트리밍.

## 앱 쪽 (1단계에서 이미 구현됨)
카드 연출 레이어(카드가 슬라이드되어 나오고, 실제 결과로 뒤집히며, 페이즈에 맞춰 타이밍)는
현재 2D 펠트 위에서 먼저 구현되어 있습니다. 딜러 영상은 2단계에서 같은 카드 레이어 뒤의
배경 레이어로 추가됩니다 — 위의 카드 슬롯 JSON이 둘을 정렬하는 기준입니다.
